# Лабораторная работа №3. Архитектура торгового розничного предприятия (C4 model + BPMN)

## Описание задания

Разработать архитектуру розничного предприятия, которое использует интернет-магазин (B2C), мобильные приложения, CRM, WMS, 1С: Торговля, 1С: Денежные средства, 1С: Документооборот.  
Необходимо:

- Описать **текущую архитектуру (AS IS)** с помощью C4 model (контекст, контейнеры).
- Спроектировать **целевую архитектуру (TO BE)** с добавлением ETL, хранилища данных (DWH) и BI-системы, а также интеграции с BrandAnalytics (мониторинг соцмедиа).
- Построить диаграмму потоков данных.
- Разработать BPMN-процессы:  
  - ETL-сбор данных из всех источников,  
  - прогнозирование закупок на основе продаж и тональности отзывов.
- Обосновать выбор типов данных в ХД.

## Архитектура (C4 model)

### 1. Текущая архитектура (AS IS)

**Контекстная диаграмма** – показывает внешние сущности (клиент, менеджер) и существующие системы предприятия, а также их связи.  
`c4_context_as_is.puml`

**Контейнерная диаграмма** – детализирует внутренние приложения и базы данных, подчёркивая разрозненность данных.  
`c4_containers_as_is.puml`

### 2. Целевая архитектура (TO BE)

**Контекстная диаграмма** – добавляет ETL, DWH, BI и внешний источник BrandAnalytics.  
`c4_context_to_be.puml`

**Контейнерная диаграмма** – раскрывает слои хранилища (Staging, Core, Mart), сервер ETL (Airflow) и BI-сервер.  
`c4_containers_to_be.puml`

### 3. Диаграмма потоков данных

Показывает логическое движение данных от источников через ETL в слои DWH и далее в BI.  
`data_flow_diagram.puml`

## BPMN-процессы

### 1. ETL: сбор данных из всех источников

Файл `etl_bpmn_to_be.bpmn` описывает:

- Параллельное извлечение из 5 источников (интернет-магазин, CRM, WMS, 1С, BrandAnalytics).
- Подпроцесс трансформации (очистка, нормализация, дедупликация).
- Последовательную загрузку в слои Staging → Core → Mart.
- Логирование и завершение.

Все элементы соединены корректными sequence flow.

### 2. Прогнозирование закупок с учётом тональности отзывов

Файл `forecast_bpmn_to_be.bpmn` описывает:

- Выбор менеджером периода и категории товаров.
- Выгрузку исторических продаж из DWH.
- Получение средней тональности отзывов за период.
- Запуск модели прогноза (линейная регрессия + ARIMA).
- Сохранение прогноза в витрину и утверждение менеджером.

## Обоснование типов данных в ХД (DWH)

| Тип данных | Примеры атрибутов | Обоснование |
|------------|-------------------|--------------|
| `INTEGER` / `BIGINT` | `id_товара`, `количество`, `id_клиента` | Быстрое индексирование, агрегации SUM/COUNT |
| `TIMESTAMP` | `дата_заказа`, `дата_отгрузки` | Необходим для временных рядов, прогнозирования |
| `DECIMAL(15,2)` | `цена`, `выручка`, `себестоимость` | Точность для финансов (избегаем ошибок float) |
| `VARCHAR(n)` | `название_товара`, `категория`, `город` | Ограниченная длина для эффективного поиска |
| `TEXT` | `технологический_стек`, `полный_отзыв` | Неструктурированные данные, поиск по ключевым словам |
| `BOOLEAN` | `флаг_возврата`, `флаг_промо` | Фильтрация, условные агрегаты |
| `FLOAT` | `коэффициент_тональности` | Только для расчётных коэффициентов |
| `JSON` | `метаданные_из_BrandAnalytics` | Гибкость для данных с переменной структурой |

**Выбор СУБД для DWH:**  
- **ClickHouse** – оптимальна для аналитики больших объёмов в реальном времени (retail).  
- **PostgreSQL** – универсальный вариант для средних нагрузок.
